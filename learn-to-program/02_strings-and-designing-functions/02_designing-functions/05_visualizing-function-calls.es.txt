En este vídeo vamos a utilizar el visualizador python para comprender cómo las llamadas a una función son gestionadas en la memoria del ordenador. Hemos escrito una función denominada Transformar en  Minutos, la cual transforma un número de horas en su equivalente en minutos. Por ejemplo, si ejecutamos Transformar en Minutos con argumento 2, se supone que obtendremos 120, ya que hay 120 minutos en dos horas. Fíjate en la sangría. Todo el código que forma parte de una función debe tener sangría, normalmente cuatro espacios. Vamos a ver qué ocurre cuando ejecutamos el programa. Cuando leemos la definición de la función o cuando lo hace Python vemos que se crea una variable llamada transformar en minutos. Contiene la dirección de memoria de un objeto función. Ese objeto función contiene toda la información sobre dicha función, incluyendo todo el código necesario para su ejecución, el parámetro y la cadena.
