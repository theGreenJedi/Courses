Programma's houden erg veel informatie bij. Sommige informatie is zichtbaar voor de gebruiker, bijvoorbeeld, waarden in een spreadsheet tabel, maar veel informatie is verborgen. Bijvoorbeeld, als een spreadsheet programma het gemiddelde berekend van een groep getallen zal het programma eerst all de getallen optellen dan tellen hoeveel getallen er zijn en daarna pas de deling doen. Al die waarden worden bijgehouden of opgeslagen in het computer geheugen. In deze cursus zullen we het computer geheugen beschouwen als een erg lange lijst van opslag plaatsen, waarvan elk een unieke nummer heeft. Deze unieke nummer noemt men een geheugen adres. Meestal zullen we geheugen adressen met een 'x' ervoor schrijven zodat ze er een beetje anders uitzien dan andere getallen. X201 is dus geheugen adres 201. Je kan deze geheugen adressen beschouwen als huisnummers in een lange straat Zoals we al eerder zeiden, worden waarden opgeslagen in het computer geheugen. Dus het getal 8.5 kan in geheugen adres x34 zitten en het nummer 44 kan in het geheugen adres x35 opgeslagen zitten. Programma's hebben een manier nodig om die waarden bij te houden. Ze doen die aan de hand van iets dat men variabelen noemt. Een variabele is een benoemde locatie in het geheugen. Python houdt variabelen bij in een ander geheugen gebied dan de variabelen. We kunnen zo dus een variabele shoe<u>size hebben die geheugen adres x34 bevat, wat betekent dat</u> shoe<u>size verwijst naar waarde 8.5. Op dezelfde manier kunnen we een variabele</u> Paul<u>Age hebben, wiens waarde 44 is. We bereiken dit door geheugen adres</u> x35 op te slaan in variabele Paul<u>age. Het leuk van Python is dat je</u> je geen zorgen moet maken over welke specifieke geheugen adressen er gekozen worden. Python zal dit bijhouden voor jou. Je kan zelfs niet eens zelf kiezen welke geheugen adressen je wil gebruiken. Het helpt wel om dit model in je achterhoofd te houden als je aan het programmeren bent.
